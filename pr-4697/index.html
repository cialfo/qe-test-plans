Regression
Communications — Notifications TC-NOTIF-001–048 Notifications and the Activity Feed page share the same /v4/feeds/activities data and ActivityFeedsService state; the new dismiss action mutates local feed state (optimistic removal, deadline count, pagination reload, in-flight reload cancellation) that Notifications TCs assume is otherwise read-only.
Dashboards & Reports — Activity Reports TC-ACTR-001–020 Activity Reports surfaces the same feed/activity data set; a dismissed item is now excluded from the underlying query (per MAN-946) and existing report TCs should be re-checked for any assumption that dismissed items still count in totals or lists.
New Surfaces
Dashboards & Reports — Activity Reports — Per-item dismiss (×) control on the Activity Feed page's feed list — hover-revealed, keyboard-reachable (Tab/Enter/Space), calls PUT /v4/feeds/activities/:id/dismiss, optimistically updates local state with rollback on failure, guards duplicate in-flight dismisses of the same item, cancels a stale in-flight reload when a different item is dismissed before the first reload resolves, and reconciles pagination via a full reload of previously-loaded pages.
