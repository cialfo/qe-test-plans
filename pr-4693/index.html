Auth — Authentication TC-AUTH-001–003, 006, 008–009, 017 SigninComponent now injects CustomAssetsService and calls loadRememberedUser()/loadLoginBanner() during tenant/subdomain resolution on every sign-in page load. A wiring break here (missing DI provider, thrown error in the subscribe path, or a change to tenant-token resolution) risks breaking the base sign-in page for every role, not just the ad path — the tenant token used to fetch the banner is the same schoolAccessToken/tenantAccessToken used by the login form itself.
Auth — Authentication TC-AUTH-007 getLoginBanner's token now travels via a Tenant-Access-Token header instead of an api_access_token URL query param — a change to how the tenant token is transmitted on a pre-auth endpoint. Worth a quick regression check that password-reset's own token handling (a separate endpoint) was not touched by this refactor.
New Surfaces
Auth — Authentication — Sponsored login-ad banner on the sign-in page's promo panel (AuthPromoPanelComponent + PublicAuthLayoutComponent + SigninComponent): for a returning user identified by a remembered email, the backend can resolve a targeted university ad (topline, up to 3 two-tone headlines, uni logo, click-through URL) that replaces the default rotating Cialfo brand copy. Fires view/click analytics + backend interaction tracking (view cap, permanent click-dismiss), and includes new client-side URL-safety and pre-auth PII handling (no tenant token or remembered email in the URL/logs).
