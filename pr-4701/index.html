Regression
App Shell (cross-cutting — no dedicated board module) N/A — global chrome app.component.ts/.html is the root shell rendered on every route. This PR adds AppUpdateService.start() to ngOnInit and mounts <app-update-banner> as a fixed, z-index:100000, bottom-center overlay in the template. It also extends the existing NavigationError branch (previously only called loadingService.finish()) to additionally call appUpdateService.onChunkError(event.error). No existing per-module TC set exercises this file, but every module's pages now render under this shell — a banner false-positive or the overlay's fixed positioning could visually collide with any bottom-fixed UI (toasts, chat widgets, mobile bottom nav) on any page, and the NavigationError change is a new side effect on a router event every module's lazy routes emit.
New Surfaces
None on board — new global app-shell surface — Self-update-check banner: AppUpdateService polls /<base-href>/version.json (visibilitychange/focus/online triggers + ~15min jittered fallback timer, 30s cooldown) and compares the remote build sha to the sha bundled at boot; on mismatch shows a non-dismissible bottom-center banner with a single Reload action. Separately, window 'error' (capture)/'unhandledrejection' listeners plus the app's router NavigationError branch detect chunk-load/dynamic-import failures (a stale-asset signal, not necessarily a new deploy) and surface a generic non-dismissible 'reload' banner. First-detected prompt is terminal — polling stops and the prompt cannot be overridden or dismissed until the user clicks Reload.
